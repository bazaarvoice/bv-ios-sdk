Overview

API for posting Custom Content to Curations.

– submitCurationsContentWithParams:completionHandler:withFailure:

This call wraps the Curations “Custom Post” API. Besides having your client ID and Curations API configured in the BVSDKManager, all you need to do is use one of the initializers in the BVCurationsAddPostRequest and make the API call.

- (void)submitCurationsContentWithParams:(nullable BVCurationsAddPostRequest *)postParams completionHandler:(nonnull uploadCompletionHandler)completionHandler withFailure:(nonnull uploadErrorHandler)failureHandler

Parameters

postParams
  • A valid initialized BVCurationsAddPostRequest object.
completionHandler
  • Only called when a custom post has succeeded. This handler contains no parameters and is invoked on the main thread.
failureHandler
  • Called when there is an API failure of some kind. Error code and error text are returned in an NSError object. This handler is invoked on the main thread.
