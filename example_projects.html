The Bazaarvoice SDK for iOS comes with Example projects in both Swift and Objective-C for each component in the SDK. The Examples are written in such a way that you can easily drop in your own configurations files and quickly get familiar with the APIs and data. There is also a full kitchen-sink style BVSDKDemo app that contains all the SDK functionality in a mock eCommerce mobile app. The BVSDKDemo app runs out-of-the-box with mock data so you can run it to get an idea of how Bazaarvoice solutions might fit into your mobile solution.

Run your own data in the Example Apps

Any of the example apps can be configured to display your own data. To do this you will need to visit the Installation and Configuration page to create and download your configuration files. Once you've entered a valid configuration you will be given the option to download your configuration files. This will download bvsdk_config_prod.json and/or bvsdk_config_staging .json depending on API keys entered. These files will need to be added to the example app target.

Basic Example Projects

Product Recommendations, Advertising, Curations, and Conversations all come with example projects, which demonstrate how to properly implement these modules. Each of the BVSDK modules below are located under the /Examples directory and have both Swift and Objective-C reference implementations:

  • Conversations: Demonstrates a minimal configuration for constructing a Conversations API request for display or content submission.
  • Curations: Shows how to display a social feed and sumbit photos.
  • Product Recommendations: Shows how you can fetch and display product recommendations for a user.
  • Advertising: Shows how you can utlize Bazaarvoice's shopper segment data to provide targeting advertising to your customers.

Running examples

The example project require the use of Cocoapods to set up the XCode workspace before you can run any of the samples.

  1. From a terminal, navigate to an SDK module under the Examples/* directory. Navigate to either the Swift or Obj-C folder.
  2. Run pod install and then open the corresponding .xcworkspace file with Xcode.
  3. Open the AppDelegate file, and set your API keys and clientId in the didFinishLaunchingWithOptions: method.

BVSDKDemo: Full SDK Integration Examples
The full BVSDK project works out-of-the box under the /Examples/BVSDKDemo folder. Just run pod install in that folder and open the BVSDKDemo.xcworkspace to execute the project. By default the app runs in mock mode (using OHHTTPSStubs) and will display sample data from a ficticious project catalog for a ficticious company, Endurance Cycles.

The demonstration shows the major components of the SDK from Product Recommendations, Advertising, Curations REST API and Custom Photo Upload. The Conversations examples demonstrate how read Conversation statistics, fetch reviews and questions as well as demonstrations on how to construct forms for uploading UGC content such as Rating & Reviews, Questions, and Feedback.

  Note: The BVSDKDemo uses App Groups capabilities and an Extension. Becuase of this you won't be able to run this app on your device unless you remove the Curations Custom Post Extension or add the proper permission to your own App ID in your own Apple Developer Account.
