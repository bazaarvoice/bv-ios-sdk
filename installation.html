Installation

This guide shows you how to install and configure the Bazaarvoice Mobile SDK into your iOS app.

Prerequisites

  • Signed Data Usage Amendment, if not already in place
  • Minimum iOS deployment target of iOS 8.0
  • Xcode version 7.0 or higher
  • Your Client ID
  • API key(s) for the Bazaarvoice product you have enabled

1 Select Configuration Options

Configure your SDK options.
The values you set will generate the necessary script and Mobile SDK intialization code to get you up and running quickly.

If you don't know your client ID or API key values, contact Client Care or your implementation team.

2 Install the SDK With CocoaPods

Configure your Podfile for CocoaPods installation.

Bazaarvoice recommends installation with Cocoapods to include the Mobile SDK in your project. If Cocoapods is not installed on your computer, refer to the Cocoapods installation guide for installation instructions.

Edit your Podfile to include the following:

# Default install: Installs Conversations and core modules.
pod 'BVSDK'

After you configure your Podfile, run pod install from your project directory to install the SDK modules. Then open the corresponding .xcworkspace file CocoaPods has created.

The Mobile SDK is now included in your project. Add the required imports to your project in the next step.

3 Configure Headers/Imports

Add the correct header file imports.

Add the correct import statements so you can reference the Mobile SDK modules you are using.


  • Swift
  • Objective-C / Swift with bridging header
// When using Dynamic Frameworks with Swift (not using Objective-C Bridging header),
// use the folling import in each of your .swift files
import BVSDK
    
// Import BVSDK.h for all modules, including Conversations
#import <BVSDK/BVSDK.h>

Manual installation is not recommended, due to difficulty of dependencies and upgrading SDK versions.
